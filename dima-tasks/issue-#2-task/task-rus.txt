	Input Numbers History Task
	

ОПИСАНИЕ
	
	Предполагается следующее поведение программы, то есть что её пользователь
	может делать:

	- вводить число, ну или любую абракадабру;
	- вводить специально зарезервировааное слово. Они бывают следующие:
		- back (или символы <=, то есть стрелка влево) означает "просмотреть
		последнюю/предидущую введённую строку". Эта строка может быть число, а
		может быть и что-то другое. То есть back означает шагнуть в истории
		введённых строк "назад";
		- forth (или символы =>, то есть стрелка вправо) означает "просмотреть
		следующую введённую строку". То есть forth означает шагнуть в истории
		введённых строк "вперёд";
	
	Таким образом, пользователь может вводить любые строки (хотя и просят его
	ввести число) и просматривать историю своего ввода: ввод слова back
	распечатывает на экране предидущую введённую команду, а forth это шаг в
	истории вперёд. Для удобства введены стрелки <= и =>, которые полностью
	дублируют поведение слов/команд back и forth соответственно. Выйти из
	программы можно если ввести exit, quit или просто q.

	Примечание.
	При просмотре истории введённых строк, бывают ситуации когда пользователь
	"утыкается в концы истории": это либо нет следующей строки, либо нет
	предидущей. В этих случаях можно выводить на экран соответствующее
	сообщение. Например, "No input".
